Vamos a practicar lo aprendido en esta sección (Api deportes).


1. En tu workspace, crear una nueva colección llamada "API deportes "

2. Dentro de tu colección, crear las siguientes requests:

1. Nombre de la request : Lista todas las ligas deportivas

GET URL: https://www.thesportsdb.com/api/v1/json/2/all_leagues.php

Por medio de API tests, crear las siguientes validaciones a la respuesta.

    1. Validar que la respuesta se procese por debjo de 800 milisegundos

    2. Validar que la respuesta contenga un texto que diga “Leagues”

    3. Validar que la respuesta contenga una liga con nombre “English League Championship”

2. Nombre de la request : Lista los equipos por deporte y país

1. GET URL: https://www.thesportsdb.com/api/v1/json/2/search_all_teams.php?s=Soccer&c=spain

Utilizando “Pre req scripts” se debe crear una variable global llamada country y generar un país aleatorio por cada ejecución.(tip: usar '{{$randomCountry}}')

Pasar como parámetro del request la variable país en el parámetro c

Por medio de API tests, crear las siguientes validaciones a la respuesta.

  • Validar que la respuesta contenta un encabezado "Content-Type" en el header

  • Validar que la respuesta no contine valores nulos en los campos IDteam, nombre del equipo (strTeam) y año de creacion (intFormedYear)


3. Nombre de la request : Lista jugadores por nombre

1. GET URL: https://www.thesportsdb.com/api/v1/json/2/searchplayers.php?p=messi

Por medio de API tests, crear las siguientes validaciones a la respuesta.

  • Validar que la respuesta contenga en el campo “strDescriptionEN” el texto “leo”

  • Valida que el campo strTeam contengta un valor tipo string
